Clone

Running this code allows you to run a local copy of a repositry and syncs them.

To use this command copy the link to the repository and paste it after typing git Clone:

git clone https://github.com/YOUR-USERNAME/YOUR-REPOSITORY

Cloning a repository pulls down a full copy of all the repository data that GitHub.com has at that point in time, including all versions of every file and folder for the project. You can push your changes to the remote repository on GitHub.com, or pull other people's changes from GitHub.com. For more information, see Using Git.

Commit

Records what changes were made, when they were made and by who.

To use commit type git commit -m"" then explain changes made in the quotation marks:

git commit -m "updated the README.md with link to contributing guide"

When you make a commit, you must include a commit message that briefly describes the changes.

Pull

Summary on command

Allows you to automatically update your local repo with the content of the remote repository

How to use it

To use this command type git pull in the terminal you are working on and press enter

git pull

More on the command

The git pull command first runs git fetch which downloads content from the specified remote repository. Then a git merge is executed to merge the remote content refs and heads into a new local merge commit.

Push

Updates the main repository with the changes made on a local one.

The git push command accepts two arguments, the remote name and the branch name:

git push REMOTE-NAME BRANCH-NAME

After you make and commit changes locally, you can share them with the remote repository using git push. Pushing changes to the remote makes your commits accessible to others who you may be collaborating with. This will also update any open pull requests with the branch that you're working on.
